# PRD：笔记列表常驻 + 技能管理

> 版本：v1.0 | 日期：2026-04-07 | 作者：NoteAI Team

---

## 1. 背景与目标

上一轮重构将 NoteAI 改为 AI-First 双栏布局 `[AI面板 | 编辑器]`，原有的 `NoteList` 组件被移出渲染树，用户只能通过标题栏浮层访问笔记列表。用户反馈**笔记列表应常驻显示**，方便快速切换笔记。同时，随着内置技能数量增加，需要提供**技能管理能力**，支持启用/禁用内置技能以及创建自定义技能。

### 目标

1. 在编辑器右侧常驻显示 280px 笔记列表，形成三栏布局
2. 新增技能管理页面，支持内置技能开关和自定义技能 CRUD
3. 自定义技能无需编写代码，通过提示词模板即可创建

---

## 2. Feature 1：右侧常驻笔记列表

### 2.1 需求描述

| 项目 | 说明 |
|------|------|
| 优先级 | P0 |
| 目标用户 | 所有 NoteAI 用户 |
| 核心价值 | 提升笔记切换效率，无需打开浮层即可浏览和切换笔记 |

### 2.2 布局设计

```
┌──────────────────────────────────────────────────────┐
│ NoteAI                        笔记▼  提醒  技能  ⚙  │
├─────────────────┬─────────────────┬──────────────────┤
│                 │                 │                  │
│   AI 对话面板    │    笔记编辑器    │    笔记列表      │
│   (flex-1)      │    (flex-1)     │    (280px)       │
│                 │                 │                  │
└─────────────────┴─────────────────┴──────────────────┘
```

### 2.3 功能详情

- 复用现有 `NoteList` 组件，从 `useNotesStore` 读取数据
- 支持笔记浏览、点击切换、新建笔记、右键删除
- 多选模式：可批量选择并删除笔记
- 笔记卡片展示：标题、预览文本、时间戳、标签
- 置顶笔记高亮显示
- NotesBrowserOverlay 浮层保留（提供文件夹筛选功能，与常驻列表互补）

### 2.4 技术方案

| 文件 | 变更 |
|------|------|
| `src/pages/Notes.tsx` | 添加 `NoteList` 为第三列，加分隔线 |
| `src/components/NoteList/NoteList.tsx` | `border-r` → `border-l`（右侧放置） |
| `electron/main.ts` | `minWidth: 900` → `1100` |

### 2.5 验收标准

- [ ] 笔记列表在右侧 280px 宽度常驻显示
- [ ] 点击笔记可切换到对应笔记进行编辑

## 3. Feature 2：技能管理页面

### 3.1 需求描述

| 项目 | 说明 |
|------|------|
| 优先级 | P1 |
| 目标用户 | 需要定制 AI 行为的进阶用户 |
| 核心价值 | 灵活控制 AI 技能，支持无代码创建自定义技能 |

### 3.2 功能详情

#### 3.2.1 内置技能管理

| 技能 | 图标 | 描述 | 默认状态 |
|------|------|------|----------|
| 图片分析 | 🖼️ | 图片描述、OCR 文字识别 | 启用 |
| 数据分析 | 📊 | CSV/JSON 统计摘要 | 启用 |
| 文档摘要 | 📄 | PDF/长文档要点提取 | 启用 |
| 翻译 | 🌐 | 多语言互译 | 启用 |
| 写作助手 | ✍️ | 润色、改写、扩写 | 启用 |

- 每个内置技能可通过开关启用/禁用
- 禁用后，AI 面板技能栏不显示该技能
- 重启应用后状态保持

#### 3.2.2 自定义技能

用户可创建自定义技能，无需编写 JavaScript 代码，只需填写：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 技能显示名称 |
| 图标 | 是 | Emoji 图标 |
| 描述 | 否 | 技能功能描述 |
| 分类 | 是 | 分析 / 写作 / 翻译 / 工具 |
| 系统提示词 | 否 | 定义 AI 角色和行为 |
| 用户提示词模板 | 否 | 支持 `{input}` 占位符 |

自定义技能支持编辑和删除。

### 3.3 页面设计

```
┌──────────────────────────────────────┐
│ 技能管理                              │
│                                      │
│ ── 内置技能 ─────────────────────     │
│ 🖼️ 图片分析    图片描述、OCR   [开关] │
│ 📊 数据分析    CSV/JSON 统计   [开关] │
│ 📄 文档摘要    PDF要点提取     [开关] │
│ 🌐 翻译        多语言互译      [开关] │
│ ✍️ 写作助手    润色改写扩写    [开关] │
│                                      │
│ ── 自定义技能 ────────────────────    │
│ 🔧 我的技能  自定义描述  [✏️][🗑️][开关]│
│ [+ 添加自定义技能]                    │
│                                      │
│ ── 添加技能（展开时） ───────────     │
│ 名称: [____]    图标: [____]         │
│ 描述: [____]                         │
│ 分类: (分析/写作/翻译/工具)           │
│ 系统提示词: [textarea]               │
│ 用户提示词模板: [textarea]           │
│ [保存] [取消]                        │
└──────────────────────────────────────┘
```

### 3.4 导航

- 标题栏新增"技能"按钮，使用 Puzzle 图标

### 3.5 数据持久化

复用 `settings` 表 `getSetting/setSetting`，存储键 `skills_config`：

```json
{
  "disabledSkillIds": ["translation"],
  "customSkills": [
    {
      "id": "custom-1712500000-abc123",
      "name": "代码审查",
      "icon": "🔍",
      "description": "审查代码质量",
      "category": "analysis",
      "systemPrompt": "你是一个代码审查专家...",
      "userPromptTemplate": "请审查以下代码：\n{input}"
    }
  ]
}
```

## 5. 风险与注意事项

1. **窗口宽度**：三栏布局需要至少 1100px，低分辨率屏幕可能需要滚动
2. **自定义技能安全**：提示词由用户自行编写，不执行任何代码，安全风险低
3. **持久化兼容**：使用现有 settings 表，无需 schema 迁移
