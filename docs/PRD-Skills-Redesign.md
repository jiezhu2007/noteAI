# PRD：内置技能重设计 & 移除冗余 QuickActions

- **版本**：v1.0
- **日期**：2026-04-07
- **状态**：已实施

---

## 1. 背景与目标

### 1.1 背景

NoteAI 当前内置 5 个 AI 技能（图片分析、数据分析、文档摘要、翻译、写作助手），提示词设计较为简单，缺乏：
- 结构化输出引导（用户获得的结果格式不统一）
- 场景细分（同一技能内的不同使用场景未区分处理）
- 输入/输出格式规范（依赖模型自由发挥）

同时，AI 面板中的"摘要"和"要点"两个 QuickActions 按钮功能已被"文档摘要"技能完全覆盖，属于冗余入口。

### 1.2 目标

1. 参考 ClawHub 技能市场设计理念，重设计 5 个内置技能的提示词
2. 移除冗余的 QuickActions 组件，简化 AI 面板

### 1.3 设计原则

- **聚焦场景**：每个技能明确定义适用场景和使用方式
- **结构化输出**：使用 Markdown 标题分层，统一输出格式
- **自适应**：根据输入内容特征自动调整处理策略
- **可预期**：用户能预期每次调用的输出结构

## 2. 功能设计

### 2.1 技能重设计

#### 2.1.1 图片识别（原：图片分析）

| 属性 | 内容 |
|------|------|
| **ID** | `image-analysis` |
| **名称** | 图片识别 |
| **图标** | 🖼️ |
| **描述** | 内容描述、OCR 文字提取、图表解读 |

**场景拆分：**
- **场景 A：通用内容描述** — 照片、插画、截图的主体/背景/色调/构图分析
- **场景 B：OCR 文字提取** — 逐区域提取文字，标注模糊内容，识别多语言
- **场景 C：图表解读** — 解读坐标轴/图例/数据标签，提取趋势和结论

**输出结构：** 图片类型 → 分析结果 → 置信度 → 补充说明

#### 2.1.2 数据分析

| 属性 | 内容 |
|------|------|
| **ID** | `data-analysis` |
| **名称** | 数据分析 |
| **图标** | 📊 |
| **描述** | 表格数据统计与可视化建议 |

**支持格式：** CSV、JSON、TSV、Markdown 表格

**输出结构（五步）：**
1. 数据概览（字段名、类型、缺失率表格）
2. 统计摘要（数值统计量表格 + 分类字段频次）
3. 趋势与模式（时间趋势、相关性、分布特征）
4. 异常检测（离群点、数据质量问题）
5. 洞察与建议（核心发现 + 可视化图表推荐）

#### 2.1.3 智能摘要（原：文档摘要）

| 属性 | 内容 |
|------|------|
| **ID** | `document-summary` |
| **名称** | 智能摘要 |
| **图标** | 📄 |
| **描述** | 文档智能摘要、要点提取与行动建议 |

**文档类型识别：** 学术论文 / 技术文档 / 新闻报道 / 会议纪要 / 其他

**长度自适应：**
- 短文（< 500 字）：简要输出
- 中等（500-3000 字）：完整四段式
- 长文（> 3000 字）：增加分章节摘要

**输出结构：** 文档类型 → 一句话概要 → 核心摘要 → 关键要点 → 行动建议

#### 2.1.4 多语言翻译（原：翻译）

| 属性 | 内容 |
|------|------|
| **ID** | `translation` |
| **名称** | 多语言翻译 |
| **图标** | 🌐 |
| **描述** | 多语言互译，支持多种翻译风格 |

**翻译风格：** 直译 / 意译 / 口语化 / 学术（自动选择或用户指定）

**翻译要求：**
- 保留段落格式、Markdown 标记
- 保留专有名词原文，首次附加译名
- 保留代码片段/URL/数字不翻译

**输出结构：** 翻译信息 → 译文 → 术语对照表 → 翻译说明

#### 2.1.5 AI 写作（原：写作助手）

| 属性 | 内容 |
|------|------|
| **ID** | `writing-assistant` |
| **名称** | AI 写作 |
| **图标** | ✍️ |
| **描述** | 润色、改写、扩写、缩写、续写、纠错 |

**操作模式：** 润色（默认）/ 改写 / 扩写 / 缩写 / 续写 / 纠错

**风格目标：** 正式 / 口语 / 学术 / 创意 / 商务

**输出结构：** 操作模式 → 目标风格 → 修改结果 → 修改对比表格 → 修改小结

### 2.2 移除 QuickActions

| 操作 | 说明 |
|------|------|
| 删除 `QuickActions.tsx` | 整个组件移除 |
| 更新 `AIPanel.tsx` | 移除 import 和 JSX 引用 |
| 保留后端 IPC | `ai:summarize` 和 `ai:extractPoints` 通道保留，维护接口完整性 |

**理由：** "摘要"和"要点"功能已被"智能摘要"技能完全覆盖，QuickActions 为冗余入口。

## 5. 风险与回退

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 新提示词过长导致 token 超限 | 响应截断 | 监控实际 token 使用，必要时精简提示词 |
| 用户习惯依赖 QuickActions | 操作路径变化 | 智能摘要技能提供超集功能，引导迁移 |
| 结构化输出不被模型严格遵循 | 格式不一致 | 提示词中使用代码块示例强化格式约束 |
