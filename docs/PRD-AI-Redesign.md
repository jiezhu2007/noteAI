## 1. 项目背景与目标

### 1.1 背景

NoteAI 是一款基于 Electron 的智能笔记应用，集成了 AI 对话、笔记编辑、提醒管理等功能。当前版本（v1.x）的 AI 面板是右侧 320px 的可选侧边栏，以笔记编辑为中心。

随着 AI 能力的持续增强，用户越来越多地依赖 AI 进行笔记创作、分析和整理。现有布局无法充分发挥 AI 的潜力。

### 1.2 目标

- **AI-First 体验**: 将 AI 对话面板提升为主交互界面，占据约 50% 屏幕空间
- **文件分析能力**: 支持图片、PDF、Excel、CSV 等本地文件的 AI 分析
- **可扩展技能系统**: 提供预置分析/写作/翻译技能，支持未来自定义扩展
- **简化导航**: 将文件夹/笔记列表移入浮层，减少常驻 UI 元素

### 4.3 交互流程

1. **笔记切换**: 点击标题栏"笔记▼" → 浮层打开 → 选择笔记 → 浮层关闭 → 编辑器加载新笔记

3. **技能使用**: 点击技能 chip（如"图片分析"） → chip 高亮 → 发送消息时自动注入技能 prompt → 再次点击取消

## 5. 技术架构概述

### 5.1 技术栈

- **前端**: React 18 + TypeScript + Vite + TailwindCSS
- **桌面框架**: Electron 35
- **编辑器**: TipTap (ProseMirror)
- **状态管理**: Zustand
- **数据库**: sql.js (WASM SQLite)
- **AI**: Ollama / Claude API / OpenAI API / Custom API

### 5.3 数据流

```
[用户输入 + 附件] → ChatInput
    → [技能处理] → SkillResult (systemPrompt + userPrompt)
    → chatStore.sendMessage()
    → IPC: ai:chatStart (含 images[] + skillSystemPrompt)
    → ai.ts: chatStream() (按 Provider 格式化多模态消息)
    → 流式 chunks → renderer → ChatMessageBubble
```

### 5.4 多模态消息格式

| Provider | 图片格式 |
|----------|----------|
| Ollama | `messages[].images: ['base64...']` |
| Claude | `content: [{type:'image', source:{type:'base64',...}}, {type:'text',...}]` |
| OpenAI | `content: [{type:'image_url', image_url:{url:'data:...'}}]` |
| Custom | 同 OpenAI 格式 |

- 附件存储在 `userData/attachments/`，使用 UUID 命名避免冲突
- 暗色/亮色主题均正常显示

## 附录 A: 风险与应对

| 风险 | 应对策略 |
|------|----------|
| 多模态 API 各 Provider 格式差异大 | 在 ai.ts 中按 Provider 独立实现消息格式化 |
| Ollama 不是所有模型都支持 vision | 尝试发送，失败时友好提示"该模型不支持图片" |
| pdf-parse 在 Electron 中兼容性 | 纯 JS 库，在 main process 中运行应无问题 |
| 大文件导致 AI Token 超限 | 文件 10MB 上限，文本截断 8000 字符 |
| 浮层焦点管理 | 复用 SearchModal 的 Escape/遮罩点击模式 |
